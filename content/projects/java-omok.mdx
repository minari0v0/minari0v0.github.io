# Java 소켓 기반 온라인 오목 게임


## 프로젝트 개요

Java 소켓 통신과 GUI 라이브러리인 Swing을 활용하여 개발한 멀티플레이어 온라인 오목 게임입니다. 단순한 오목 게임 구현을 넘어, 실시간 채팅, 랭킹 시스템, 사용자 정보 관리, 관리자 기능 등 다양한 부가 기능을 통해 완성도 높은 데스크톱 애플리케이션을 구현하는 것을 목표로 했습니다.

### 핵심 기능

- **실시간 멀티플레이**: 소켓(TCP/IP) 통신을 통해 여러 사용자가 동시에 접속하여 다른 사람과 오목을 두거나 관전할 수 있습니다.
- **게임방 시스템**: 난수 코드를 이용한 비공개 방 생성, 비밀번호 설정 기능으로 친구와 비공개로 게임을 즐길 수 있습니다.
- **다양한 채팅 기능**:
    - 로비/게임방 채팅: 여러 사용자와 자유롭게 대화할 수 있는 전체 채팅 및 게임방 내 채팅 기능을 제공합니다.
    - 1:1 채팅: 특정 사용자를 우클릭하여 개인적인 대화를 나눌 수 있습니다.
    - 채팅 부가기능: 이모티콘, 채팅 테마 변경, 대화 내용 검색 기능을 통해 사용자 경험을 향상시켰습니다.
- **커뮤니티 및 정보**:
    - 랭킹 시스템: 승패 기록을 기반으로 랭킹을 산정하고 금, 은, 동메달로 시각화하여 보여줍니다.
    - 날씨 API 연동: 공공 데이터 API를 활용하여 로그인 화면과 로비에서 지역별 날씨 정보를 제공합니다.
    - 사용자 프로필: 회원가입 시 설정한 프로필 이미지(오목알로 사용)와 전적 등 상세 정보를 확인할 수 있습니다.
- **관리자 기능**:
    - 회원 관리: 관리자용 GUI를 통해 전체 회원 정보를 조회, 추가, 수정, 삭제할 수 있습니다.
    - 채팅 모니터링 및 관리: 실시간 로비 채팅 내용을 확인하고, 부적절한 메시지를 삭제하거나 복구할 수 있습니다.

## 기술 스택

### Backend & Core
- **Java**: 프로젝트의 핵심 언어로, 객체 지향 프로그래밍(OOP) 원칙에 따라 전체 시스템을 설계했습니다.
- **Java Socket (TCP/IP)**: 클라이언트와 서버 간의 실시간 양방향 통신을 구현하여 멀티플레이 환경을 구축했습니다. 각 클라이언트의 요청을 안정적으로 처리하기 위해 멀티 스레딩 방식을 적용했습니다.
- **MySQL**: 사용자 정보, 게임 전적, 채팅 메시지 등 모든 데이터를 저장하고 관리하는 데이터베이스로 사용했습니다.

### Frontend
- **Java Swing**: 자바의 표준 GUI 라이브러리를 사용하여 사용자가 상호작용할 수 있는 데스크톱 애플리케이션 화면을 구성했습니다. `JPanel`, `JFrame`, `JScrollPane` 등 다양한 컴포넌트를 커스터마이징하여 사용자 경험을 개선했습니다.

### API & ETC
- **공공 데이터 포털 (날씨 API)**: 사용자의 위치에 따른 실시간 날씨 정보를 가져와 화면에 표시하는 데 사용했습니다.
- **Lombok**: `@Getter`, `@Setter`, `@Data` 등의 어노테이션을 활용하여 DTO/VO 클래스의 보일러플레이트 코드를 줄이고 코드 가독성을 높였습니다.

## 데이터베이스 설계

MySQL을 사용하여 게임의 핵심 데이터를 관리했으며, 주요 테이블 구조는 다음과 같습니다.

- **user / user_bin**: 사용자의 계정 정보(ID, PW), 닉네임, 전적(승/패), 프로필 이미지 등을 저장합니다. `user_bin`은 탈퇴한 회원의 정보를 백업하는 용도입니다.
- **chat_messages / chat_messages_bin**: 사용자들이 나눈 채팅 메시지의 내용, 발신자, 수신자, 시간 등을 기록합니다. `chat_messages_bin`은 관리자에 의해 삭제된 메시지를 보관하여 복구할 수 있도록 설계되었습니다.
- **ranking**: `user` 테이블의 전적 데이터를 바탕으로 주기적으로 순위를 갱신하고, 승률, 총 게임 수 등의 정보를 저장하여 랭킹 조회 시 빠른 응답이 가능하도록 했습니다.

## 프로젝트 성과 및 배운 점

### 기술적 성장
- **소켓 통신에 대한 깊은 이해**: TCP/IP 기반의 소켓 프로그래밍을 통해 저수준(Low-level) 네트워크 통신 원리를 학습하고, 클라이언트-서버 아키텍처를 직접 설계하고 구현하는 경험을 쌓았습니다.
- **OOP 설계 능력 향상**: 역할과 책임에 따라 클래스를 분리(e.g., `Client`, `Server`, `Room`, `DBManager`)하고, 인터페이스를 활용하여 각 기능의 결합도를 낮추는 등 객체 지향 설계 원칙을 적용하려 노력했습니다.
- **DB 설계 및 관리**: ERD 설계를 통해 데이터의 정규화와 무결성을 고려했으며, 삭제/백업 정책(`user_bin`, `chat_messages_bin`)을 통해 데이터 관리의 안정성을 높였습니다.
- **GUI 프로그래밍**: Java Swing을 다루며 이벤트 리스너, 커스텀 컴포넌트(`CustomScrollBarUI`) 등 GUI 애플리케이션의 동작 원리를 이해할 수 있었습니다.

### 문제 해결 경험
- **동시성 문제 해결**: 여러 클라이언트가 동시에 서버에 접속하고 요청을 보낼 때 발생할 수 있는 동시성(Concurrency) 문제를 해결하기 위해 `synchronized` 키워드를 사용하여 스레드 안전성(Thread-safe)을 확보했습니다.
- **프로토콜 설계**: 클라이언트와 서버 간의 원활한 통신을 위해 메시지 타입을 정의하고(e.g., `LOGIN|userID`, `CHAT|message`), 이를 파싱하여 처리하는 자체 프로토콜을 설계했습니다. 이를 통해 다양한 요청을 체계적으로 관리할 수 있었습니다.
- **관전자-플레이어 전환 로직**: 게임 진행 중 플레이어가 퇴장했을 때, 관전자 중 가장 먼저 입장한 사용자를 새로운 플레이어로 승격시키는 로직을 구현하며 복잡한 상태 변화를 관리하는 능력을 길렀습니다.

## 프로젝트 회고

이번 프로젝트는 Java의 기본적인 문법을 넘어 네트워크, 데이터베이스, GUI 등 애플리케이션 개발에 필요한 여러 기술을 종합적으로 활용해 볼 수 있었던 소중한 경험이었습니다. 특히 순수 Java 코드로 소켓 서버를 구축하며 컴퓨터 네트워크의 동작 원리를 몸소 체감할 수 있었던 점이 가장 큰 수확이었습니다.

물론 아쉬운 점도 남습니다. 현재 구조는 서버에 모든 로직이 집중되어 있어 사용자가 많아지면 성능 저하가 발생할 수 있습니다. 만약 프로젝트를 개선한다면, Spring Boot와 같은 프레임워크를 도입하여 보다 안정적이고 확장성 있는 서버를 구축하고, 웹 기반의 UI/UX를 제공하여 접근성을 높여보고 싶습니다. 또한, 비밀번호를 암호화하여 저장하는 등 보안적인 측면을 더 강화할 필요성을 느꼈습니다.

이 프로젝트를 통해 얻은 지식과 경험을 바탕으로, 앞으로 더 견고하고 사용자 친화적인 소프트웨어를 만드는 개발자로 성장해 나가고 싶습니다.
